## 1. 개요

- **한 줄 정의**: 중앙 디스플레이 영역에 떠다니는 장식용 파티클 애니메이션
- **위치**: `.stage` 내부, `.particles` 컨테이너 (z-index: 2)
- **구현 상태**: ✅ 구현 완료

## 2. UI 구조

### 2.1. 와이어프레임

```
┌─── .stage (1920x1080) ──────────────────────────┐
│                                                   │
│   ┌─── .particles (absolute, inset:0) ─────────┐ │
│   │                                             │ │
│   │     ·          ·                            │ │
│   │          ·              ·                   │ │
│   │   ·               ·          ·              │ │
│   │        ·     ·                    ·         │ │
│   │                    ·       ·                │ │
│   │     ·                          ·            │ │
│   │                                             │ │
│   └─────────────────────────────────────────────┘ │
│                                                   │
└───────────────────────────────────────────────────┘
     각 · = .particle (2px 원형, gold-soft 색상)
```

### 2.2. CSS 클래스 구조

```
.particles#particles         ← 컨테이너 (absolute, inset:0, z-index:2, pointer-events:none)
└── .particle (다수)         ← 개별 파티클 (absolute, 2px 원형)
```

### 2.3. 시각 요소 상세

#### 컨테이너 (`.particles`)

| 속성 | 값 |
|------|----|
| 포지션 | `absolute`, `inset: 0` |
| z-index | `2` |
| 상호작용 | `pointer-events: none` |

#### 개별 파티클 (`.particle`)

| 속성 | 값 |
|------|----|
| 크기 | `2px x 2px`, `border-radius: 50%` |
| 색상 | `var(--gold-soft)` |
| 초기 투명도 | `opacity: 0` |
| 그림자 | `0 0 6px rgba(232,200,124,0.5)` |
| 애니메이션 | `drift 14s linear infinite` |

#### 애니메이션 (`@keyframes drift`)

| 키프레임 | 속성 |
|----------|------|
| `0%` | `opacity: 0`, `translateY(18px)` |
| `15%` | `opacity: 0.7` |
| `85%` | `opacity: 0.4` |
| `100%` | `opacity: 0`, `translateY(-40px)` |

- 총 이동거리: 아래에서 위로 약 58px (`18px → -40px`)
- 주기: 14초, 선형 반복

## 3. 데이터 모델

### 3.1. 전역 상태

해당 없음. 파티클은 전역 상태(`A`)를 사용하지 않는다.

### 3.2. 데이터 스키마

해당 없음. 파티클은 DOM 요소로만 존재한다.

## 4. 동작 규칙

### 4.1. 상태 전이

상태 전이 없음. 페이지 로드 후 애니메이션만 반복한다.

### 4.2. 핵심 로직

- 페이지 로드 시 `#particles` 컨테이너 안에 다수의 `.particle` 요소를 **프로그래밍 방식으로 생성**
- 각 파티클은 랜덤한 `left`, `top` 위치와 `animation-delay` 값을 부여받아 시차 애니메이션을 구현
- CSS `@keyframes drift`가 상향 이동 + 페이드인/페이드아웃을 처리

### 4.3. 함수 매핑

| 함수 | 역할 |
|------|------|
| (초기화 코드) | 페이지 로드 시 `.particle` 요소를 동적 생성하여 `#particles`에 삽입 |

## 5. 사용자 상호작용

### 5.1. 조작 방법

없음. 순수 장식용 컴포넌트로 `pointer-events: none`이 적용되어 있다.

### 5.2. 키보드 단축키

해당 없음.

### 5.3. 이벤트 흐름

```
페이지 로드 → 파티클 DOM 생성 → CSS drift 애니메이션 무한 반복
```

## 6. 관련 시스템

| 시스템 | 관계 |
|--------|------|
| `sigil` | 같은 `.stage` 내에 z-index 3으로 파티클 위에 표시 |
| `focus-main-display` | 파티클이 배경으로 깔리는 중앙 디스플레이 영역 |
