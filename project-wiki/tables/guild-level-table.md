## 4. 현재 하드코딩 값 (Lv.1 기준)

| 항목 | 값 | 코드 변수 | 해금 연동 |
|------|-----|----------|----------|
| 길드원 보유 상한 | 8명 | `A.guildMemberCap = 8` | — (v2에서 고정) |
| 모집 확률 weight | 80/20/0/0 | `RARITY_BY_LEVEL_V2` | O (레벨별 bracket) |
| 스쿼드 슬롯 수 | 2명 | `SQUAD_SLOTS = 2` | O (squad3/squad4 구매) |
| 스쿼드 프리셋 수 | 1개 | `PRESET_COUNT = 1` | O (preset2/preset3 구매) |
| 보상 배수 | ×1.0 | `getRewardMultiplier()` | O (5단계 구매) |
| 기본 제공 트랙 | 2곡 (idx 0,1) | `isTrackUnlocked()` | O (콘텐츠 구매) |

## 6. 레벨 곡선 설계 원칙

| 원칙 | 설명 |
|------|------|
| **지수 성장** | 100 × 1.4^(level-1), 후반부 급격 증가 |
| **골드 비용 선형** | 500 × level, XP보다 완만한 브레이크 |
| **수동 레벨업** | 자동 레벨업 없음 — EXP+골드 충족 후 클릭 |
| **디자인 캡 20** | Lv.20 이후 해금/효과 없음 (숫자만 증가) |
| **카테고리 해금** | 레벨 조건 없이 골드로 자유 구매 (requires 체인) |
