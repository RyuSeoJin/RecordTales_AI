## 1. 해결하려는 문제

- 할 일 관리 도구는 **지속 사용 동기**가 약하다 — 오늘 처리하면 끝, 내일 다시 열 이유가 약함
- 현재 프로토타입의 길드 레벨(`Lv.7`)은 **정적 더미**이므로 의미가 없음
- 사용자가 **노력의 축적**을 시각적으로 체감할 장치가 필요

## 2. 아이디어

### 2.1. 기본 방향

**RPG 길드 성장** 은유. 퀘스트 완료·Focus 세션 누적으로 XP를 획득하고 길드 레벨이 상승한다.

### 2.2. XP 획득 경로 (초안)

| 행동 | XP (예시) |
|------|---------|
| 일반 퀘스트 완료 | +10 |
| 반복 퀘스트 인스턴스 완료 | +5 |
| Focus 세션(25분) 완료 | +15 |
| Focus 세션(연속 3회) 보너스 | +20 |
| 주간 완료율 70% 이상 | +50 |
| 주간 완료율 100% | +100 |

※ 수치는 **전부 예시**. 밸런싱은 별도 작업.

### 2.3. 레벨 곡선

- Lv.1 → Lv.2: 낮은 장벽 (초기 동기 부여)
- 이후 지수 곡선으로 증가
- 레벨 상한: 미정 (무한 vs 상한 논의 필요)

### 2.4. 레벨 업 보상 (아이디어)

- 시각 보상: 시길 애니메이션, 파티클 연출
- 기능 해금 가능성:
  - Lv.3: 길드원 모집 시스템 해금?
  - Lv.5: 스쿼드 편성 해금?
  - Lv.10: 시길·테마 커스터마이징 해금?

## 3. 예상 UI (대략적)

### 3.1. `daily-status-panel` (좌상) 내 표시

```
┌────────────────────────────┐
│  [G]  길드명                │
│       Lv.3  ████████░░ 64% │   ← XP 게이지 추가
│       Guild Master         │
└────────────────────────────┘
```

### 3.2. 레벨 업 순간 연출

- 중앙 `focus-main-display`에서 시길이 한 번 빛나며 커지는 애니메이션
- 상단 토스트: "Lv.4 달성 — [해금 내용 있으면 함께 표시]"

## 4. 열린 질문

- [ ] **XP 공식 확정**: 각 행동별 XP 수치 — 밸런싱 기준?
- [ ] **레벨 곡선**: 선형 vs 지수 vs 단계적 — 어느 감각이 맞나?
- [ ] **레벨 상한**: 무한 레벨 vs 상한(예: Lv.99)
- [ ] **해금 시스템**: 기능 해금으로 레벨에 의미 부여 vs 순수 시각 보상만
- [ ] **XP 차감 여부**: 퀘스트 지연·포기 시 XP 차감? (부정적 강화 회피 vs 현실성)
- [ ] **연속 스트릭**: 연일 달성 보너스 개념 도입?
- [ ] **주간·월간 집계**: 리셋 주기 및 보너스 구조
- [ ] **HTML 프로토타입 반영 여부**: 현 프로토타입 단계에서 간이 구현 vs 언리얼 전환 후로 미룸
- [ ] **초기 Lv.1 기간**: 얼마나 빨리 Lv.2에 도달해야 "성장 체감"이 생기나?
- [ ] **시각 연출 강도**: 게임적 화려함 vs 집중 도구의 "과한 화려함 지양" 원칙 균형
