## 1. 기획 의도

- 생산성 도구에 **수집형 요소**를 추가하여 게이미피케이션을 강화
- 집중(Focus/Stopwatch)·퀘스트 완료로 획득한 재화(골드)를 소모해 길드원을 수집하는 **보상 루프** 구축
- 길드원 모집을 시작으로, 능력치·팀 편성·시너지 등 후속 시스템의 진입점 역할

## 2. 아이디어

### 2.1. 기본 방향

**길드원 모집** 시스템. 골드를 소모하여 모집을 진행하면 3명의 길드원 후보가 랜덤으로 나타나고, 유저가 그 중 1명을 선택하거나 패스하여 다음 기회를 기다린다.

### 2.2. 모집 비용

- **1회 모집 비용**: 1,000골드
- 골드 획득 경로(집중 세션 적립, 퀘스트 보상)와 상세 경제 규칙은 `gold-economy` (approved) 참조

### 2.3. 모집 흐름

```
[길드원 모집] 버튼 선택
      ↓
  골드 1,000 차감
      ↓
  3명의 길드원 후보 카드 공개 (등급별 차등 연출)
  (등급·능력치 전부 공개)
      ↓
  유저가 1명 선택 → 획득
  또는 패스 → 골드 환불 없음, 모집 종료
```

### 2.4. 후보 카드 정보

- **공개 범위**: 등급 + 능력치 전부 공개 (선택 전 비교 가능)
- 카드에 표시되는 정보:
  - 길드원 이름
  - 등급 (Normal / Rare / Epic / Legendary)
  - 능력치 목록 및 수치
  - 시각 에셋 (카드 이미지)

### 2.5. 선택과 패스

- **선택**: 3명 중 1명을 선택하면 즉시 획득
- **패스**: 3명 모두 마음에 안 들면 패스 가능. **골드 환불 없음** — 이미 소모된 1,000골드는 돌아오지 않음
- 보유 상한 도달 시 모집 버튼 비활성화 (상한은 `guild.md`에서 정의)

### 2.6. 등급 체계

- 4단계: Normal(★) / Rare(★★) / Epic(★★★) / Legendary(★★★★)
- 등급별 속성(능력치 개수·수치 범위)은 `guild-member` (approved) 참조
- **출현 확률**: 길드 레벨에 비례하여 가중치 변동 (구체적 확률 테이블은 미정)

### 2.7. 모집 연출

#### 2.7.1. 연출 타임라인

```
T+0ms     "모집하기" 클릭 → 골드 차감, 후보 3명 생성
          모집 공고서 바람에 날려감 (0.8s)

T+800ms   중세 문 등장 (석벽 + 양쪽 횃불)
          쿵... 쿵... 쿵... (노크 3회, 400ms 간격)
          → 화면 미세 흔들림(shake) + 충격파 파티클

T+2000ms  양쪽 문이 열림 (1.2s, 3D perspective rotateY)
          문 틈새로 빛 쏟아짐 (빛 색상 = 최고 등급 색)

T+3000ms  카드 3장 순차 등장
          문 너머에서 걸어나오듯 아래→위 슬라이드 (200ms 간격)

T+3600ms  순차 플립 시작 (300ms 간격)
          등급별 이펙트 (glow, 파티클, flash)

T+4500ms  스탯 + 선택/패스 버튼 활성화
```

#### 2.7.2. 등급별 연출

| 등급 | 카드 플립 연출 | 문빛 색상 | 추가 이펙트 |
|------|--------------|----------|------------|
| Normal ★ | 단순 플립 | 흰색 `#f0ede8` | — |
| Rare ★★ | 플립 + glow | 파란색 `#7cb8e8` | `box-shadow 0 0 24px` |
| Epic ★★★ | 플립 + glow + 파티클 | 보라색 `#a87ce8` | `box-shadow 0 0 24px` + 파티클 |
| Legendary ★★★★ | 플립 + glow + 파티클 + 전체 flash | 주황색 `#e8a05c` | `box-shadow 0 0 28px` + 지속 발광 + 파티클 |

#### 2.7.3. 카드 레이아웃

카드 내부는 이미지(55%) + 등급 + 이름만 표시. 능력치와 선택 버튼은 카드 **외부 하단**에 배치하여 능력 개수에 따른 레이아웃 찌그러짐 방지.

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│  (이미지)  │  │  (이미지)  │  │  (이미지)  │
│   55%    │  │   55%    │  │   55%    │
├──────────┤  ├──────────┤  ├──────────┤
│ ★ Normal │  │ ★★★ Epic │  │ ★★ Rare  │
│  Aldric  │  │  Fiora   │  │   Kael   │
└──────────┘  └──────────┘  └──────────┘
  집중+5%      집중+12%      퀘보+8%      ← 카드 외부
               골드+10%
  [선택]        [선택]        [선택]       ← 카드 외부
```

#### 2.7.4. 선택/패스 마이크로 애니메이션

| 동작 | 연출 | 시간 |
|------|------|------|
| 선택 | 선택 카드 `scale(1.08)` + 금색 glow, 나머지 카드 페이드아웃 + 슬라이드다운 | 600ms 후 idle |
| 패스 | 전체 카드 페이드아웃 + 슬라이드다운 | 400ms 후 idle |

### 2.8. 배반 가챠 (Betrayal Gacha)

모집 연출 중 "기대를 뒤집는" 도파민 연출. 처음에 한 단계 낮은 등급으로 보여주다가 빛이 폭발하며 진짜 등급이 드러난다.

#### 2.8.1. 발동 조건

| 조건 | 값 |
|------|-----|
| 대상 | 후보 3명 중 **최고 등급**이 Rare 이상일 때 |
| 확률 | **50%** |
| 제외 | Normal (아래 등급이 없으므로 발동 불가) |

> 확률은 기존 모집 확률과 **독립**. 등급 결정은 이미 완료된 상태에서 **연출만** 변경.

#### 2.8.2. 배반 조합

| 실제 등급 | 처음 보여주는 등급 (속임) | 전환 연출 |
|-----------|------------------------|----------|
| Rare ★★ | Normal 흰빛 | → 폭발 → 파란빛 |
| Epic ★★★ | Rare 파란빛 | → 폭발 → 보라빛 |
| Legendary ★★★★ | Epic 보라빛 | → 폭발 → 주황빛 |

#### 2.8.3. 연출 타임라인

```
일반 연출과 동일하게 진행 (노크 → 문 열림)

T+1200ms  문 열림 → 속임 등급 색상 빛 표시
          (예: Legendary인데 Epic 보라빛)

T+1900ms  빛 폭발(betrayalFlash) + 화면 흔들림 3회
          16~24개 방사형 파티클 (진짜 등급 색)
          빛 색상 전환 (속임 → 진짜)
          빛 크기 순간 확장 (180px → 220px) 후 수축

T+3000ms  카드 등장 (일반보다 ~800ms 지연)
          이후 일반 연출과 동일
```

#### 2.8.4. 코드 매핑

```javascript
// startHire() 내 배반 판정
const bestTier = rarityOrder[best.rarity];       // 최고 등급 티어 (0~3)
const betrayal = bestTier >= 1 && Math.random() < 0.5;  // 50% 확률
const fakeRarity = betrayal ? rarityTiers[bestTier - 1] : best.rarity;
const lightColor = colors[fakeRarity];            // 문빛에 속임 색 적용
```

| 함수/변수 | 역할 |
|-----------|------|
| `betrayal` (boolean) | 배반 가챠 발동 여부 |
| `fakeRarity` | 처음 보여줄 속임 등급 |
| `.hire-betrayal-flash` | 빛 폭발 전체 화면 플래시 |
| `.hire-betrayal-particle` | 방사형 파티클 (진짜 등급 색) |
| `@keyframes betrayalFlash` | 플래시 확장 → 페이드아웃 |
| `@keyframes betrayalBurst` | 파티클 방사 → 페이드아웃 |

#### 2.8.5. 설계 원칙

- **확률 독립**: 등급 확률 시스템에 영향 없음. 순수 연출 레이어.
- **하위 호환**: 배반 미발동 시 기존 연출 그대로 진행.
- **도파민 설계**: "아 Rare인가…" → "Epic이었어!" 반전 쾌감. 발동 빈도 50%로 빈번한 서프라이즈 제공.
