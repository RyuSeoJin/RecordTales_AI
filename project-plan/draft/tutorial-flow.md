## 1. 해결하려는 문제

- 최초 실행 시 사용자가 **길드명을 설정**해야 한다 (현재 프로토타입은 `Grimhollow` 고정)
- 사용자가 **세계관(길드 마스터 은유)에 자연스럽게 진입**해야 한다 — 단순 설명 모달이면 몰입감이 약함
- 주요 기능(Focus / Quests / Archive / Command Post)의 존재를 **간단히 인지**시켜야 한다 — 긴 튜토리얼은 집중 도구의 철학과 맞지 않음

## 2. 아이디어

### 2.1. 기본 방향

**짧고 서사적**인 튜토리얼. 3~5 스텝 이내로 끝내고 "길드 마스터 임명" 같은 세계관적 프레이밍으로 감쌈.

### 2.2. 예상 스텝 (초안)

1. **임명**: "당신은 새 길드의 마스터로 임명되었습니다" — 연출 + 환영
2. **길드명 설정**: 사용자가 원하는 길드명 입력. 기본 제안값 제공 가능
3. **첫 퀘스트 작성**: 안내와 함께 첫 할 일을 하나 작성해 보도록 유도 (옵션)
4. **Sigil 계시**: 중앙 `focus-main-display`의 시길이 사용자 길드명으로 갱신되는 연출
5. **완료**: "Lv.1 Guild Master — Per focum, ad gloriam" 메시지

### 2.3. 트리거 조건

- 저장소(로컬 스토리지)에 길드 정보가 없을 때
- 사용자가 설정에서 "튜토리얼 다시 보기"를 선택했을 때

## 3. 예상 UI (대략적)

```
┌──────────────────────────────────────────────┐
│                                              │
│        [시길 애니메이션 중앙 배치]              │
│                                              │
│     "당신은 새 길드의 마스터로..."              │
│                                              │
│     ┌─────────────────────────────┐         │
│     │  길드명 입력: [_________]    │         │
│     └─────────────────────────────┘         │
│                                              │
│                       [Continue →]           │
└──────────────────────────────────────────────┘
```

- 전체 화면 오버레이로 일반 UI를 잠시 가림
- 배경에 파티클·시길은 유지되어 연속성 부여
- 각 스텝은 페이드 전환

## 4. 열린 질문

- [ ] **스킵 가능 여부**: 스킵 버튼 제공? 아니면 최소 1~2 스텝은 강제?
- [ ] **스텝 수 확정**: 3스텝 vs 5스텝 — 어느 쪽이 집중 도구 철학과 맞나?
- [ ] **언리얼 전환 시점**: 현 HTML 프로토타입 단계에서도 구현? 아니면 언리얼 전환 후에만?
- [ ] **재생 조건**: 설정에서 "다시 보기" 제공 여부
- [ ] **길드명 변경**: 튜토리얼 이후에도 설정에서 변경 가능해야 하는가?
- [ ] **기본 제안값**: 랜덤 판타지 이름 제안? 아니면 빈 필드에서 시작?
- [ ] **첫 퀘스트 작성 스텝**: 강제 vs 옵션 vs 생략
- [ ] **다국어 지원**: 한국어/영어 동시 제공 필요성
