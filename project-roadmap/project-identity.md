## 1. 한 줄 정의 (Elevator Pitch)

> **RecordTales**는 길드 마스터가 자신의 길드를 운영하듯,
> 사용자가 스스로의 하루를 기록·관리하는 **데스크톱 집중 도구**입니다.

판타지 길드 운영 세계관을 UI 은유로 사용하되, 본질은 **할 일 관리 + 집중 타이머 + 기록 보관** 도구입니다.

## 2. 제품 정체성

### 2.1. 제품 개요

| 속성 | 값 |
|------|-----|
| **이름** | RecordTales |
| **최종 형태** | 언리얼 엔진 기반 데스크톱 애플리케이션 |
| **현재 단계** | HTML 디자인 프로토타입 (`guild-master-desktop.html`) |
| **해상도** | 1920×1080 고정 |
| **플랫폼** | PC (언리얼 엔진 기반) |
| **저장소** | 설치한 사용자의 **로컬 스토리지** |
| **네트워크** | 현재 없음. 향후 **Steam 연동** 여지 (업적·클라우드 세이브 등) |

### 2.2. 대상 사용자

- **1인 작업자**: 혼자 집중 작업을 하는 프리랜서·개발자·크리에이터·학생
- **루틴 관리자**: 반복 작업과 일회성 작업을 구분해서 관리하고 싶은 사람

### 2.3. 형태 — 현재 단계

- **최종 형태**: 언리얼 엔진 기반 데스크톱 애플리케이션
- **현재 단계**: HTML 디자인 프로토타입 (`guild-master-desktop.html`)

즉 최종 개발은 언리얼 엔진으로 진행되지만, 현재는 **디자인·UX 검증을 위한 HTML 프로토타입**으로 구현되고 있습니다. HTML 프로토타입의 작업 제약·시각 디자인 규칙은 `html-design-style-guide.md`에서 전담 정의합니다.

## 3. 세계관 (Worldview)

### 3.1. 판타지 길드 은유

사용자의 일상 작업을 **길드 운영**으로 재프레이밍합니다.

| 현실 | RecordTales 은유 |
|------|-----------------|
| 사용자 | 길드 마스터 |
| 하루의 할 일 | 오늘의 퀘스트 |
| 반복 작업 | 연쇄 퀘스트 (반복 마스터가 매일 발행) |
| 완료한 일의 기록 | 아카이브 (길드 연대기) |
| 집중 시간 | 출정 (Focus 세션) |
| 진행 중인 작업 | 중단(suspended) 가능한 퀘스트 |

### 3.2. 기본 길드 설정

| 항목 | 최종 방향 | 현재 프로토타입 표시 |
|------|---------|---------------------|
| **길드명** | 최초 실행 시 **튜토리얼에서 사용자가 직접 설정** | `Grimhollow` (디자인 확인용 더미) |
| **길드 레벨** | **Lv.1부터 시작**, 경험치 시스템으로 상승 | `Lv.7` (디자인 확인용 더미) |
| **사용자 역할** | Guild Master (고정) | Guild Master |

현재 프로토타입에 표시된 `Grimhollow / Lv.7`은 **레이아웃·타이포그래피 확인을 위한 더미**이며, 최종 제품에서는 튜토리얼을 통해 사용자가 설정한 길드명과 Lv.1 초기 상태로 시작합니다.

## 4. UX 원칙 (UX Principles)

사용자 경험 설계 시 **반드시** 지키는 원칙입니다. HTML 프로토타입·언리얼 최종본 **모두에 적용**됩니다.

| 원칙 | 뜻하는 바 | 실제 반영 |
|------|---------|----------|
| **집중 우선** | 화면 중앙은 항상 비어 있거나 집중 대상만 표시 | 기능 UI는 4개 코너로 분산 배치 |
| **과한 화려함 지양** | 눈을 피로하게 만들지 않는다 | 어두운 글래스 모피즘, 절제된 애니메이션 |
| **필요한 순간만 강조** | 평소엔 조용하고, 상태 변화 시에만 시각적 반응 | 지연 배지, 완료 체크 순간의 마이크로 인터랙션 |
| **키보드 친화** | 마우스 없이도 주요 흐름이 완결되어야 함 | `Escape`, `Enter`, `Shift/Ctrl+Enter` 규칙 |
| **은유 → 기능** | 세계관이 먼저지만, 기능이 흐려지면 은유를 양보한다 | "Quests" 외에 "할 일"을 공식 용어로 병용 |

## 5. 미래 방향성 (Roadmap Vision)

### 5.1. 단기 — 리팩터 ~ 현 프로토타입 완성

**핵심 리팩터 (확정됨)**:

- 데이터 모델 재작성: 반복 인스턴스를 **실제 물리 객체**로 저장
- 상세 뷰 **인라인 편집** 도입
- 반복 제거 새 동작: `recEndDate` 변경 → **인스턴스 물리 삭제**
- 자정 롤오버 재작성

상세 로드맵은 `project-plan/approved/quests-panel.md` (예정) 참조.

### 5.2. 중기 — Draft 시스템 구현

현재 `project-plan/draft/`에 아이디어 단계로 존재하는 시스템들:

| 시스템 | 역할 | 예상 위치 |
|--------|------|---------|
| **길드원 고용** | 가상의 "길드원"에게 반복 작업 분담·의인화 | `guild-menu-panel` (우하) |
| **스쿼드 편성** | 관련 퀘스트를 묶는 단위 | `guild-menu-panel` (우하) |
| **환경 설정** | 테마·배경음 볼륨·시각 효과 제어 | `guild-menu-panel` (우하) |

draft → approved 승격은 **개발 착수 결정 시점**에 이루어집니다.

### 5.3. 장기 — 언리얼 전환 및 그 이후

- **언리얼 엔진 기반 재구현**: HTML 프로토타입에서 검증된 레이아웃·UX·세계관 톤을 언리얼 엔진으로 이식
- **로컬 스토리지 영속성**: 사용자 설치 환경의 로컬 디스크에 상태 저장
- **Steam 연동 검토**: 업적(achievements), 클라우드 세이브, 프렌즈·통계 등
- **통계·회고 뷰**: 주간·월간 완료 패턴 시각화
- **세션 시각화 강화**: `focus-main-display` 영역에서 Focus 세션 진행을 시각적으로 표현
- **길드 커스터마이징 확장**: 시길·모토·테마 사용자 설정

**관련 draft 시스템** (별도 파일로 분리 관리):
- `project-plan/draft/tutorial-flow.md` — 최초 실행 시 길드명 설정·세계관 안내
- `project-plan/draft/experience-system.md` — 퀘스트 완료·세션 누적 기반 레벨링 (Lv.1 시작)
- `project-plan/draft/guild-member-hiring.md` — 가상 길드원 시스템
- `project-plan/draft/squad-formation.md` — 퀘스트 묶음 단위
- `project-plan/draft/environment-settings.md` — 테마·볼륨 등 설정

※ 이 항목들은 **확정된 계획이 아닌 탐색 영역**입니다. 승격 결정 시 `approved/`로 이동합니다.
